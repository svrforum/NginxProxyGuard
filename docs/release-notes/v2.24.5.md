## 🛡️ 전역 신뢰 IP가 이제 개별 호스트 보안까지 우회합니다

전역 신뢰 IP(설정 → 보안)가 그동안 자동 차단(Fail2ban·WAF 자동 차단·IP 차단)과 속도 제한에는 적용됐지만, **개별 프록시 호스트의 지역 제한(차단 모드)·접근 목록·클라우드 차단**에는 적용되지 않아 호스트마다 IP를 다시 입력해야 했습니다. 이제 신뢰 IP가 이 **경계 보안들도 일괄 우회**합니다.

- 호스트가 10개여도 전역 신뢰 IP 한 번 설정으로 모든 호스트에 적용됩니다.
- 보안을 위해 **WAF 규칙과 익스플로잇/스캐너 차단은 계속 적용**됩니다 — 신뢰 IP라도 공격 트래픽을 백엔드로 전달할 수 없습니다.

## ☁️ 프록시 호스트가 만든 DDNS 레코드의 'Cloudflare 프록시' 설정

프록시 호스트에서 자동 등록된 DDNS 레코드는 그동안 'Cloudflare 프록시'를 켤 수 없었습니다. 이제 가능합니다.

- DDNS 페이지에서 관리형 레코드의 **프록시·TTL·활성 여부를 직접 편집**할 수 있습니다(호스트명/프로바이더는 호스트가 소유하므로 잠금).
- 프록시 호스트 편집 화면에 **'Cloudflare 프록시' 기본값** 토글이 추가되어, 이 호스트에서 새로 생성되는 DDNS 레코드가 기본값을 상속합니다. 레코드별로 따로 바꾼 값은 그대로 유지됩니다.

## 🔧 Challenge 모드에서 발생하던 nginx 설정 오류 수정

일부 환경(구버전 docker-compose, Synology 등)에서 Challenge 모드를 켜면 `nginx -t` 가 `the closing bracket in "API_HOST_PORT" variable is missing` 오류로 실패하던 문제를 수정했습니다.

- 원인은 docker-compose의 중첩 변수 기본값이 일부 파서에서 그대로 남는 것이었습니다. 이제 API 주소를 `API_HOST_PORT` 기준으로 안전하게 도출하고, 잘못된 값이 설정에 들어가지 않도록 방어합니다.
- 커스텀 API 포트를 쓰는 경우도 올바르게 동작합니다.
