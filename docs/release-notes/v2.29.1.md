## 🩹 로그 통계에서 내부 헬스체크(PipelineCanary) 제외 (#191)

내부 로그 파이프라인 점검용 요청(PipelineCanary)이 **로그 통계에 집계되던** 문제를 고쳤습니다.

### 🐛 수정
- **PipelineCanary 요청이 통계에서 제외 (#191)**: NginxProxyGuard는 로그 수집 파이프라인이 정상 동작하는지 5분마다 내부 점검 요청(`/__npg_canary`)을 보냅니다. 이 요청은 사용자 로그·통계에서 제외되어야 하는데, 일부 통계 집계 경로(`로그 → 통계`의 상위 User-Agent·국가·요청 수 등)에서 빠지지 않아 `Go-http-client` 같은 내부 트래픽이 통계에 잡혔습니다.
  - 이제 **모든 통계 집계에서 일관되게 제외**됩니다. 점검 요청 데이터 자체는 파이프라인 검증에 필요하므로 보관되며, 화면 표시에서만 제외됩니다.
  - 대시보드의 대표 수치(총 요청 수 등)는 이전에도 정상이었고, `로그 → 통계` 화면의 상위 목록에서 주로 보이던 문제입니다.

**Full Changelog**: https://github.com/svrforum/NginxProxyGuard/compare/v2.29.0...v2.29.1
