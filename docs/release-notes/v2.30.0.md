## 🔐 접속 로그에서 쿼리스트링 제외 옵션 (#195)

API 키·액세스 토큰·웹훅 시크릿을 **쿼리 파라미터로 전달하는 앱** 때문에 이런 값들이 접속 로그(및 회전·압축된 아카이브)에 평문으로 남는 문제를 해결하는 옵션을 추가했습니다.

### ✨ 새 기능
- **"접속 로그에서 쿼리스트링 제외" 토글 (설정 → 전역 설정 → 고급)**: 켜면 접속 로그에 **경로(path)만** 기록하고 쿼리스트링(`?apiKey=…`, `?access_token=…` 등)은 남기지 않습니다.
  - 예: `GET /api/image?apiKey=SECRET` → `GET /api/image` 로만 기록.
  - **기본값은 꺼짐**(현행 유지) — 필요한 사용자만 켜면 됩니다.
  - method·status·host·클라이언트 IP·User-Agent·차단 사유 등 분석에 필요한 정보는 그대로 유지됩니다.

### 📝 참고
- 내부 로그 파이프라인 점검(PipelineCanary)은 이 옵션과 무관하게 정상 동작하도록 처리되어 있습니다.
- 옵션을 켜면 nginx 설정이 자동 재생성·reload됩니다.
