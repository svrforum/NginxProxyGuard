## 🛡️ 버그 수정

대시보드 보안 통계가 표시되지 않고 DB 로그에 `out of shared memory`, `unsupported Unicode escape sequence`가 반복되던 문제를 수정했습니다. (#152)

- **chunk 락 고갈 수정**: 로그/통계 쿼리의 시간 필터를 `timestamp` → `created_at`(TimescaleDB 파티션 키)으로 변경하여 chunk pruning을 복구. 모든 chunk에 락을 잡아 `max_locks_per_transaction`을 초과하던 문제 해소.
- **jsonb NUL 삽입 오류 수정**: `system_logs` 삽입 단계에서 NUL/유니코드 널 이스케이프를 정화하여 jsonb 거부로 인한 시스템 로그 유실 방지.
- **하이퍼테이블 retention 추가**: `logs_partitioned`/`system_logs`/`audit_logs` chunk를 `drop_chunks`로 정리. 기존 정리 로직은 TimescaleDB chunk를 지우지 못해 chunk가 무한 증가했음.
- `max_locks_per_transaction=512` 방어 마진 추가.

> ℹ️ 참고: 로그/통계의 시간 범위 필터가 이벤트 시각(`timestamp`)이 아닌 수집 시각(`created_at`) 기준으로 동작합니다. 실시간 수집 환경에서는 사실상 동일합니다.
