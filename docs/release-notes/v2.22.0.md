## 🚀 새로운 기능

### Stream(TCP) 호스트 TLS 종단 (#156)

Stream 호스트에서 **TLS를 종단(terminate)** 한 뒤 평문으로 백엔드에 전달할 수 있습니다. 기존에는 L4 패스스루(`ssl_preread`)만 가능해, 평문 백엔드(예: 내부 HTTP 포워드 프록시)를 HTTPS 엔드포인트로 노출할 수 없었습니다.

- 프록시 호스트 폼(Stream) → **TLS 모드** 선택:
  - **없음** — 평문 TCP
  - **TLS 패스스루** — `ssl_preread` (기존 동작, SNI 기반 패스스루)
  - **TLS 종단** — 인증서를 선택하면 게이트웨이가 TLS를 종단하고 평문으로 백엔드에 전달
- TLS 종단은 TCP 전용이며 패스스루와 상호 배타입니다. 인증서 미발급 시 발급 전까지 자동으로 평문 처리되어 reload 실패를 막습니다.
- Stream 백엔드 입력에서도 **Docker 컨테이너 찾아보기**를 사용할 수 있습니다.

생성되는 설정 예:
```nginx
server {
    listen 18888 ssl;
    ssl_certificate     /etc/nginx/certs/<cert-id>/fullchain.pem;
    ssl_certificate_key /etc/nginx/certs/<cert-id>/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    proxy_pass <backend>;
}
```

### Docker 선택기에 관리 UI(npg-ui) 노출 (#155)

프록시 호스트의 **Docker 컨테이너 선택기**에 NPG 관리 UI(`npg-ui`)가 표시됩니다. 이제 관리 UI를 도메인·SSL과 함께 외부로 노출하기가 쉬워졌습니다. (DB·캐시·내부 API 등 다른 NPG 인프라 컨테이너는 오선택 방지를 위해 계속 숨겨집니다.)

**전체 변경 이력:** https://github.com/svrforum/NginxProxyGuard/compare/v2.21.0...v2.22.0
