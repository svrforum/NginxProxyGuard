## 🚀 새로운 기능

### 프록시 호스트 ↔ DDNS 통합 (#157)

프록시 호스트와 DDNS를 하나로 묶었습니다. 이제 호스트마다 DDNS 자동 등록을 켜기만 하면, 그 호스트의 도메인이 공인 IP 변경에 맞춰 자동으로 갱신됩니다.

- **호스트별 DDNS 자동 등록 토글** — 프록시 호스트 추가/수정 화면의 *기본* 탭에서 "DDNS 자동 등록"을 켜고 DNS 프로바이더(Cloudflare / DuckDNS)를 선택하면, 해당 호스트의 모든 도메인이 관리형 DDNS 레코드로 자동 생성됩니다. 내부 IP를 가리키는 호스트가 의도치 않게 공인 IP로 노출되지 않도록 기본은 꺼짐(옵트인)입니다.
- **호스트 변경 자동 동기화** — 호스트의 도메인을 추가·삭제하거나 DDNS를 끄면 관리형 레코드도 함께 갱신됩니다. 호스트를 삭제하면 관리형 레코드도 자동으로 정리됩니다(원격 DNS 레코드는 건드리지 않습니다).
- **프록시 호스트에서 일괄 추가** — *설정 → DDNS* 페이지의 "프록시 호스트에서 추가" 버튼으로 기존 호스트 여러 개를 골라 한 번에 DDNS를 켤 수 있습니다.
- **갱신 주기 설정** — *설정 → DDNS* 에서 스케줄러가 공인 IP를 확인·갱신하는 주기(분)를 직접 지정할 수 있습니다(최소 1분).
- DDNS 페이지에서 프록시 호스트가 관리하는 레코드는 "프록시 연동" 배지로 표시되며, 호스트 편집 화면에서 관리하도록 직접 편집/삭제는 비활성화됩니다.

## 🛡️ 안정성 / 버그 수정

- **백업 복구 시 DDNS 갱신 주기 보존** — 백업/복구 라운드트립에 DDNS 갱신 주기 설정이 포함되지 않아 복구 후 기본값(5분)으로 되돌아가던 문제를 수정했습니다.
- **UI 편집 시 DDNS 동기화 보장** — 프록시 호스트를 화면에서 저장할 때 도메인 변경·DDNS 토글 변경이 관리형 DDNS 레코드에 즉시 반영되도록 수정했습니다.
