## ⚡ 로그 통계 "상위 URI" → "상위 경로"로 최적화

로그 화면의 상위 URI 통계가 대용량 환경에서 가장 느리던 부분이었는데, **경로 기준 집계**로 바꿔 빠르고 더 유용하게 개선했습니다.

- 캐시버스터 쿼리스트링(예: `?t=1778768329`) 때문에 URL이 요청마다 거의 고유해져, 상위 URI 집계가 매우 느렸습니다(대용량 환경에서 ~3초). 이제 **쿼리스트링을 제외한 경로 기준**으로 묶어 집계해 **약 3.6배 빨라집니다**.
- 같은 경로의 캐시버스터 변형들이 하나로 합쳐져, **"상위 경로"** 결과가 훨씬 의미 있습니다(차트 이름도 "상위 경로 / Top Paths"로 변경).
