## 🚀 새로운 기능

### 내장 DDNS Updater (#154)

공인 IP가 바뀌면 등록한 호스트네임의 DNS A 레코드를 자동으로 갱신합니다. 별도 DDNS 컨테이너 없이 NPG 안에서 관리할 수 있습니다.

- **설정 → DDNS** 에서 호스트네임 + DNS 프로바이더를 등록하면, 공인 IPv4 변경 시 수 분 내 A 레코드가 자동 갱신됩니다.
- 지원 프로바이더: **Cloudflare, DuckDNS** (기존 DNS 프로바이더 자격증명을 그대로 재사용). *Dynu는 추후 추가 예정.*
- 마지막 IP·동기화 시각·상태(성공/오류)를 UI에서 확인하고, "지금 갱신"으로 수동 동기화할 수 있습니다.

### 프록시 호스트 복제 시 Docker 컨테이너 선택 + 목록 정렬 (#153)

- **복제 모달에서 Docker 컨테이너를 선택**할 수 있습니다(생성 폼과 동일). 복제 후 곧바로 다른 컨테이너를 가리키게 하기 쉬워졌습니다.
- **Docker 컨테이너 선택 목록이 결정적으로 정렬**됩니다(컨테이너 이름 / 네트워크 / 포트 번호순). 기존에는 새로고침할 때마다 포트·네트워크 순서가 뒤바뀌어 sftpgo처럼 포트가 많은 서비스를 찾기 어려웠습니다.

## 🛡️ 안정성 / 버그 수정

- **신규 설치 마이그레이션 신뢰성**: 새 테이블의 외래 키 순서 문제로 fresh install이 실패할 수 있던 점을 수정(설치/업그레이드 모두 검증).
- 복제 시 컨테이너 바인딩이 일관되게 처리되도록 수정(수동으로 host를 바꿔도 낡은 바인딩이 남지 않음).
- 릴리스 워크플로의 릴리스 노트 생성 단계가 일부 조건에서 실패하던 문제 수정.
