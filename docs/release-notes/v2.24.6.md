## 🔒 보안 강화 릴리즈 — 업그레이드 권장

전체 코드베이스 보안 점검에서 확인된 항목들을 수정했습니다. 인터넷에 노출해 운영하는 경우 특히 업그레이드를 권장합니다.

### 인증 · 권한
- **초기 설정(기본 계정) 보호**: 자격 증명을 변경하기 전에는 서버 측에서 보호 엔드포인트 접근을 차단합니다. 이제 기본 계정으로는 자격 증명 변경 외에 아무 작업도 할 수 없습니다.
- **API 토큰 권한 적용**: 각 API 토큰의 권한 범위(읽기/쓰기/삭제 등)가 실제로 엔드포인트별로 강제됩니다. (이전에는 유효한 토큰이면 범위와 무관하게 모든 API를 호출할 수 있었습니다.)
- **신뢰 프록시 IP 처리**: 로그인 잠금·API 토큰 IP 허용목록·요청 제한·감사 로그가 실제 클라이언트 IP를 기준으로 동작하도록 보정했습니다(헤더 위조로 우회 불가).

### 입력 검증 · 설정 주입 방지
- 프록시 호스트 **수정** 시에도 전달 호스트/도메인 형식을 검증해, 조작된 값으로 nginx 지시자를 주입하는 것을 차단합니다.
- 고급 설정(advanced_config)에서 위험한 지시자 차단 범위를 넓혔습니다.

### 웹 보안
- 로그 화면에서 발생할 수 있던 **저장형 XSS를 제거**하고, 관리 UI에 **Content-Security-Policy**를 추가했습니다.

### 백업 · 네트워크
- 백업 복원 시 **경로 탐색(디렉터리 이탈) 방지** 및 개인키 파일 권한 보정.
- 클라우드 제공자 IP 목록 조회의 **SSRF 차단**(내부/메타데이터 주소 접근 거부).

### 컨테이너 하드닝
- `no-new-privileges`, 불필요한 Linux capability 제거, UI 컨테이너 비루트 실행.

### 버그 수정
- **인증서 → DDNS / DNS 프로바이더 목록 페이지네이션** 추가 — 20개를 넘는 항목도 페이지 이동으로 모두 조회할 수 있습니다. (#162)
