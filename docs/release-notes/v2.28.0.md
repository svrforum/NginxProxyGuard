## 🐳 인증 제공자 도커 컨테이너 선택 + IP 자동 추적 + 상태 관측성

인증 제공자(ForwardAuth)의 인증 서비스 주소를 IP로 직접 입력하지 않고 **실행 중인 도커 컨테이너에서 선택**할 수 있고, 컨테이너 IP가 바뀌면 자동으로 갱신합니다. 더불어 그 자동 추적이 **잘 되고 있는지/문제가 있는지를 화면과 로그에서 바로 확인**할 수 있게 했습니다(#181).

### ✨ 새 기능 — 도커 컨테이너 선택 (#181)
- **컨테이너에서 선택**: `인증 → 인증 제공자 → 추가/편집`의 인증 서비스 URL 옆 **"도커 컨테이너에서 선택"** 버튼으로, 실행 중인 컨테이너와 네트워크·포트를 골라 주소를 자동으로 채웁니다(프록시 호스트의 컨테이너 선택과 동일한 방식). "연결 해제"로 언제든 수동 입력으로 되돌릴 수 있습니다.
- **IP 변경 자동 추적**: 선택한 컨테이너의 IP가 바뀌면 백그라운드에서 다시 확인해 인증 제공자 주소를 갱신하고, 그 제공자를 사용하는 모든 프록시 호스트의 설정을 다시 생성합니다(#150/#151과 동일한 안전 동작 — 멀티 네트워크는 선택한 네트워크 기준으로만 추적).

### 📊 새 기능 — 인증 제공자 상태 표시 (#181)
- **상태 배지**: 컨테이너 기반 인증 제공자 카드에 **🟢 동기화됨(현재 IP·시각)** / **🔴 컨테이너 못 찾음**(툴팁에 원인) 배지를 표시합니다. "왜 인증이 안 되지?"를 컨테이너 로그를 뒤지지 않고 한눈에 파악할 수 있습니다.
- **로그 기록**: 컨테이너 IP가 바뀌거나 컨테이너를 찾지 못하면 **로그 화면(시스템 로그)**에 기록됩니다. 백업을 컨테이너가 없는 다른 환경에 복원한 경우에도 곧바로 🔴로 드러납니다.
- **Fail-closed 원칙**: 인증 서버가 다운돼도 호스트의 인증 보호는 절대 자동 해제되지 않습니다(문제만 표면화, 보호는 유지).

### 🐛 수정
- **인증 제공자 선택 콤보박스**: 프록시 호스트 보안 탭에서 제공자가 있는데도 첫 항목이 "인증 제공자가 없습니다…"로 뜨던 문제 수정 → 제공자가 있으면 **"사용 안 함"**으로 표시.

### 🔧 안정성
- **SSL 미적용 호스트에 Authelia/Authentik 연결 차단**: 이들은 보안 쿠키 특성상 보호 대상이 https여야 하므로, SSL이 꺼진 호스트에 연결하면 인증이 조용히 실패하던 것을 저장 단계에서 **"SSL을 먼저 켜세요"** 메시지로 막습니다(커스텀 제공자는 영향 없음).
- **재조정 견고화**: IP 갱신 중 nginx 설정 검증이 실패하면 주소를 이전 값으로 되돌려 다음 주기에 재시도(DB·nginx 영구 불일치 방지).
- **연결 해제 시 주소 초기화** / **자가서명 https 검증 서버 대응**(verify 위치에 `proxy_ssl_verify off`) / **도커 목록 조회 타임아웃**(느린 도커 소켓에서 무한 대기 대신 "Docker 응답 없음").
- **우회 경로 인라인 검증**: 저장 전 잘못된 경로를 즉시 표시. **삭제 시** 영향받는 호스트 수를 먼저 알리고 프록시 호스트 목록을 갱신.

### 📝 안내
- 컨테이너 선택·자동 추적은 NPG가 도커 소켓에 접근할 수 있어야 동작합니다(프록시 호스트의 컨테이너 선택과 동일 조건).
- Authelia/Authentik으로 보호할 호스트는 **SSL을 활성화**해 두세요.
- 참고: NPG에 SSL을 적용하지 않고 외부에서 TLS를 종단하는 프록시(예: Cloudflare Flexible) 뒤에 두는 구성은 별도 지원이 필요합니다(추후).
