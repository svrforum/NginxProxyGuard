## 🔐 ForwardAuth(인증 제공자) 지원 + 메뉴 전반 UI 리디자인

Authelia·Authentik 같은 외부 인증(SSO) 게이트를 프록시 호스트 앞단에 손쉽게 붙일 수 있는 **인증 제공자** 기능을 추가했습니다. 더불어 목록·폼 화면 전반을 카드 기반의 정돈된 디자인으로 새로 다듬었습니다.

### ✨ 새 기능 — 인증 제공자 (ForwardAuth) (#179)

외부 인증 서버로 접근을 먼저 검증한 뒤에만 백엔드로 트래픽을 흘려보내는 인증 게이트(nginx `auth_request`)를 GUI에서 구성할 수 있습니다.

- **재사용 가능한 "인증 제공자" 엔티티**: 인증 서버를 한 번 등록해두고 여러 프록시 호스트에서 공유합니다. 제공자 설정을 바꾸면 이를 사용하는 모든 호스트의 nginx 설정이 자동으로 다시 생성됩니다.
- **프리셋 + 커스텀**:
  - **Authelia** / **Authentik** — 검증된 연동 설정이 내장되어 서버 주소만 입력하면 동작합니다.
  - **커스텀** — 검증 경로·전달 헤더·공개(콜백) 경로·로그인 리다이렉트 등을 직접 지정해 다른 ForwardAuth 호환 서버에도 연결할 수 있습니다.
- **호스트 전체 보호 + 우회 경로**: 인증 제공자를 적용하면 해당 호스트 전체가 인증으로 보호됩니다. 인증 없이 통과시킬 경로(예: `/api`, `/health`)는 **우회 경로**로 지정할 수 있습니다.
- **적용 현황 한눈에**: `인증 제공자` 페이지에서 각 제공자가 **적용된 호스트 / 미적용 호스트**를 바로 확인하고, 그 자리에서 연결·해제할 수 있습니다.
- **적용 방법**: `프록시 호스트 → 수정 → 보안` 탭에서 인증 제공자를 선택합니다. 인증 제공자 페이지 상단의 안내에서도 위치를 알려줍니다.

### 🎨 UI 리디자인

- **카드 기반 목록 + 아이콘 액션**: 리다이렉트, 인증서(목록·이력), DNS 제공자, 접근 목록, 차단 IP, Fail2ban, 익스플로잇 규칙, 인증 제공자 등 목록 화면을 일관된 카드 디자인과 아이콘 버튼으로 정리했습니다.
- **폼 정돈**: 생성·수정 폼과 프록시 호스트의 각 탭 입력 요소를 더 깔끔하고 일관되게 다듬었습니다.
- **모달 반응성 개선**: 모달의 무거운 배경 블러를 제거하고 열림 애니메이션을 최적화해, 모달을 띄우거나 입력할 때의 버벅임을 줄였습니다.
- **로딩 깜빡임 제거**: 프록시 호스트 등에서 수정 모달을 처음 열 때 잠깐 보이던 하얀 로딩 화면을 없앴습니다.

### 🔧 안정성 · 보안

- **입력값 검증 강화**: 인증 제공자의 서버 주소·커스텀 설정 필드·우회 경로 등 nginx 설정에 반영되는 값을 사전에 검증해, 잘못된 형식이나 설정을 깨뜨릴 수 있는 문자가 들어오면 저장 단계에서 거부합니다(HTTP 400). nginx 설정 검사(`nginx -t`)도 최종 안전장치로 그대로 동작합니다.
- **백업 호환**: 인증 제공자와 호스트의 인증 연결 정보가 백업 내보내기/가져오기에 포함됩니다.

### 📝 안내

- 인증 제공자 적용·변경은 프록시 호스트 설정이 다시 생성된 뒤 반영됩니다.
- **Authelia**는 보안상 검증 대상이 `https`여야 합니다(최신 Authelia의 nginx 연동 기준). 인증으로 보호할 호스트는 SSL이 적용된 상태를 권장합니다.
- 인증 제공자는 한 호스트에서 **지역 차단(GeoIP) 챌린지(CAPTCHA) 모드와 동시에 사용할 수 없습니다.** 두 기능이 같은 위치에서 충돌하므로, 동시에 설정하면 저장이 거부됩니다.
- 사용자/그룹·역할 기반 세부 인가는 인증 서버(IdP) 쪽 정책에서 관리합니다. NPG는 "인증 통과 여부"를 게이트하고, 인증 서버가 내려준 사용자 정보 헤더를 백엔드로 전달합니다.
