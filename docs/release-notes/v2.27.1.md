## 🔑 ForwardAuth 인증이 거부되던 문제 수정 (Authelia/Authentik)

v2.27.0에서 추가된 인증 제공자(ForwardAuth)에서, Authelia가 인증 요청을 `insecure scheme 'http'` 오류로 거부하던 문제를 수정했습니다(#180).

### 🐛 수정
- **ForwardAuth 인증 요청의 scheme 처리 (#180)**: NPG가 인증 서버(Authelia/Authentik)로 보내는 검증 요청의 `X-Original-URL`이 항상 연결 scheme(`$scheme`)을 그대로 사용해, 인증 검사가 HTTP(80) 컨텍스트에서 실행되면 `http://...`로 전달되어 Authelia가 거부했습니다.
  - **SSL이 적용된 호스트는 이제 `X-Original-URL`에 `https`를 사용**합니다. 인증 검사가 HTTP(80) 블록에서 실행되거나(예: SSL은 켜고 "HTTPS 강제"는 끈 경우), NPG가 TLS를 종단하는 상위 프록시 뒤에서 동작하는 경우에도 인증 서버가 올바른 scheme을 받습니다.
  - Authelia와 Authentik 양쪽에 동일하게 적용했으며, 로그인 후 되돌아오는 리다이렉트 주소도 `https`로 교정됩니다.

### 📝 안내
- Authelia/Authentik은 세션 쿠키 보안상 보호 대상 URL이 **https**여야 합니다. 인증 제공자를 적용하는 프록시 호스트는 **SSL을 활성화**해 두세요.
- 참고: NPG에 SSL을 적용하지 않고 외부에서 TLS를 종단하는 프록시(예: Cloudflare Flexible) 뒤에 두는 구성은, 인증 서버가 요구하는 https 조건을 만족하지 못해 별도 지원이 필요합니다(추후 개선 예정).
