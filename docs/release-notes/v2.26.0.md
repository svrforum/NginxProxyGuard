## 🛡️ 전역 신뢰 IP의 WAF 우회 옵션 추가

신뢰하는 출처(내부망·모니터링·사내 서비스 등)의 정상 트래픽이 WAF(ModSecurity) 오탐으로 차단될 때, 해당 IP에 한해 WAF 검사를 건너뛰도록 선택할 수 있는 옵션을 추가했습니다.

### ✨ 새 기능
- **전역 신뢰 IP — WAF(ModSecurity) 우회 (선택, 기본 꺼짐) (#166)**: `설정 → WAF 자동 차단 → 전역 신뢰 IP` 카드에 토글이 추가되었습니다. 켜면 신뢰 IP 목록의 주소에 대해 ModSecurity/OWASP CRS 검사를 완전히 건너뜁니다.
  - **WAF만 우회합니다.** nginx 레벨의 익스플로잇/스캐너 차단과 봇 필터는 계속 적용됩니다.
  - **IP 한정**으로만 동작합니다 — 목록에 없는 IP는 종전과 동일하게 WAF 보호를 받습니다.
  - 기본값은 **꺼짐**이며, 켜면 "해당 IP에 대해 WAF 보호가 비활성화된다"는 경고가 표시됩니다. 정말로 신뢰할 수 있는 주소에만 사용하세요.

### 🔧 안정성
- **백업 복원 정확도**: 백업을 복원하면 전역 신뢰 IP 관련 설정(속도 제한 예외·WAF 우회 규칙)이 곧바로 nginx 설정에 반영되도록 했습니다(이전에는 다음 재시작까지 반영되지 않을 수 있었음).
- **설정 적용 견고화**: 메인 nginx 설정 재생성의 롤백 스냅샷을 잠금 구간 안에서 캡처해, 동시 저장 상황에서도 안전하게 이전 상태로 복구됩니다.

### 📝 안내
- WAF 관련 설정 변경은 프록시 컨테이너가 재시작된 뒤 적용됩니다(ModSecurity 규칙 특성). 토글을 켠 뒤 곧바로 반영되지 않으면 프록시를 재시작하세요.
- 신뢰 IP는 연결 출처(TCP 주소) 기준으로 판별합니다. 신뢰 프록시(예: 클라우드 프론트) 뒤에 둔 경우, 신뢰 프록시 범위 설정이 올바른지 확인하세요.
