## 🌍 GeoIP 업데이트 실패 진단성 개선 (#184)

GeoIP 데이터베이스 업데이트가 실패할 때 **원인을 알 수 있게** 하고, 가장 흔한 실패 원인을 사전에 막습니다.

### 🐛 수정
- **업데이트 실패 시 실제 에러 표시 (#184)**: 그동안 업데이트 이력의 "세부정보"에 `geoipupdate version 7.0.1 (...)` 같은 **버전 배너만** 보이고 정작 실패 원인은 잘려서 안 보였습니다. 이제 geoipupdate의 **실제 에러 메시지**(예: 인증 실패/401, 권한 없음 등)를 표시합니다.
- **MaxMind Account ID 검증**: geoipupdate 7.x는 **라이선스 키와 함께 숫자 Account ID가 반드시 필요**합니다. Account ID가 비었거나 잘못된 경우(0·비숫자), 이제 업데이트 시 **"Account ID를 설정하세요"**라는 명확한 메시지로 바로 안내합니다.
  - 라이선스 키만 재발급해도 안 고쳐지던 실패가 대부분 이 경우였습니다. `설정 → GeoIP`에서 MaxMind **Account ID(숫자)**를 함께 입력해 주세요.

### 📝 안내
- MaxMind GeoLite2는 **Account ID + License Key 두 가지 모두** 필요합니다. 둘 다 MaxMind 계정에서 확인할 수 있습니다.
