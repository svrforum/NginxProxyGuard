## 🛡️ 안정성·보안 강화 릴리즈 — 업그레이드 권장

전체 코드베이스를 점검해 신뢰성·성능·보안 전반을 다듬은 릴리즈입니다. 특히 **v2.24.6에서 컨테이너를 재생성하면 프록시 설정 저장이 실패하던 문제**를 수정했으므로, v2.24.6 사용자는 업그레이드를 권장합니다.

### 🚨 중요 수정 (v2.24.6 회귀)
- **컨테이너 하드닝으로 인한 설정 저장 불가 수정**: v2.24.6에서 API 컨테이너를 재생성하면 nginx 설정 파일 쓰기가 권한 오류로 실패하던 문제를 해결했습니다. (신규 설치·업그레이드 모두 정상 동작)

### 🔧 안정성
- **설정 적용 신뢰성**: nginx 설정 재생성을 단일 잠금 + 스냅샷/롤백으로 묶어, 일부 설정이 실패해도 이전 상태로 안전하게 복구됩니다. 적용 실패 시 호스트에 오류 상태가 표시됩니다.
- **액세스 리스트 즉시 반영**: 액세스 리스트를 수정/삭제하면 이를 사용하는 모든 프록시 호스트에 곧바로 반영됩니다(이전에는 다음 저장 때까지 옛 규칙이 유지될 수 있었음).
- **IP 차단 반영 보장**: 차단/해제 시 nginx 재적용이 일시적으로 실패하면 자동 재시도하고, 끝내 실패하면 로그로 알립니다.
- **인증서 생애주기**: 재시작 중 멈춘 발급/갱신 상태를 부팅 시 복구하고, 갱신 실패 시 백오프하며, 사용 중인 인증서·DNS 프로바이더 삭제를 차단합니다.
- **부팅 복원력**: DB가 늦게 뜨면 재시도, Valkey(캐시)가 늦게 떠도 자동 재연결, 종료 시 진행 중 요청을 정상 드레인합니다.
- **WAF 규칙 갱신**: 이미지 업그레이드 시 OWASP CRS 규칙이 최신으로 갱신됩니다(운영자가 직접 튜닝한 파일은 보존).

### ⚡ 성능
- **로그 수집 고속화**: 대량 로그 적재를 COPY 방식으로 전환해 처리량을 크게 높였습니다(대규모 환경에 효과적).
- **대시보드 부하 감소**: 통계 집계에 캐싱을 적용해 새로고침마다 무거운 재계산이 발생하지 않습니다. CPU·Docker 통계 조회도 캐싱·배치 처리합니다.
- **GeoIP 조회 최적화**: 로그 한 줄마다 발생하던 불필요한 캐시 왕복을 제거했습니다.
- **nginx 설정 효율화**: 호스트별로 중복 생성되던 필터 구독 IP 트리·공통 설정을 공유 방식으로 전환하고, 직접 포워딩 호스트에 백엔드 연결 재사용(keepalive)을 추가했습니다.

### 🔒 보안
- **고급 설정 검증 개선 (#129)**: 고급 설정의 커스텀 `location` 블록 안에서 `proxy_pass`를 다시 사용할 수 있습니다. 동시에, 관리되는 보호 영역을 벗어나 별도 서버 블록을 주입하는 시도와 로컬 파일 노출을 일으키는 지시자는 계속 차단합니다.
- **취약점 신고 체계**: 저장소에 `SECURITY.md`(비공개 취약점 신고 절차)와 의존성 자동 점검을 추가했습니다.
- **백업 복원 정확도**: 복원 시 인증서 파일을 실제로 생성하고, 보안 관련 설정 컬럼을 빠짐없이 복원하며, 손상된 백업을 완료로 표시하지 않습니다.
- **운영 가시성**: 로그인 실패 시도가 감사 로그(클라이언트 IP 포함)에 남습니다.
