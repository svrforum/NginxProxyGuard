## 🩹 목록 UI 레이아웃 안정화 + 접근 목록 견고성

연결 정보가 많은 항목에서 화면이 길게 늘어나던 문제들을 마저 정리하고, 접근 목록 조회 안정성을 높였습니다.

### 🐛 수정
- **프록시 호스트 목록 — 도메인 많은 행 캡**: 와일드카드/다중 도메인(SAN) 호스트처럼 한 호스트에 도메인이 많을 때 행이 비정상적으로 길어지던 문제를 고쳤습니다. 이제 도메인을 처음 3개만 보여주고 나머지는 **"+N개 더보기"/"접기"**로 펼칠 수 있습니다. (v2.28.3에서 고친 인증서 카드와 동일한 방식)
- **접근 목록 카드 — 항목 많을 때 스크롤**: 접근 목록을 펼쳤을 때 IP/CIDR 규칙이 많으면 카드가 끝없이 길어지던 것을, **카드 안에서 스크롤**되도록 바꿨습니다.
- **접근 목록 조회 견고성**: 설명(description)이 비어 있는(NULL) 접근 목록·규칙이 있을 때 목록 조회가 실패(500)하던 잠재 문제를 수정했습니다. (정상 경로로는 거의 발생하지 않지만, 수동 편집/구버전 데이터에서 발생 가능)
