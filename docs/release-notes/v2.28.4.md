## 🔐 nginx 보안 업데이트 + 목록 UI 안정화

nginx를 보안 패치 버전으로 올리고, 연결 정보가 많은 항목에서 화면이 길게 늘어나던 문제들을 마저 정리했습니다.

### 🔐 보안
- **nginx 1.30.1 → 1.30.2**: `ngx_http_rewrite_module` 버퍼 오버플로우 취약점(**CVE-2026-9256**)을 수정한 nginx 안정(stable) 버전으로 업데이트했습니다. `rewrite`/`return`/`if` 등 리다이렉트·force-https 처리에 쓰이는 핵심 모듈이라 권장 업데이트입니다. (mainline 1.31.x가 아닌 안정 브랜치의 보안 패치를 적용)

### 🐛 수정
- **Authentik 인증 제공자 — nginx 설정 에러 수정 (#189)**: Authentik(및 "큰 버퍼" 커스텀 제공자)를 붙인 프록시 호스트에서 `proxy_busy_buffers_size must be less than ...` 에러로 **`nginx -t`가 실패해 설정이 적용/reload되지 않던** 문제를 고쳤습니다. ForwardAuth 템플릿이 버퍼 풀(`proxy_buffers`)을 전역 설정보다 작게 덮어써 전역 `proxy_busy_buffers_size`와 충돌한 것이 원인으로, 큰 인증 응답 헤더에 실제로 필요한 `proxy_buffer_size`만 키우도록 수정했습니다.
- **프록시 호스트 목록 — 도메인 많은 행 캡**: 와일드카드/다중 도메인(SAN) 호스트처럼 한 호스트에 도메인이 많을 때 행이 비정상적으로 길어지던 문제를 고쳤습니다. 이제 도메인을 처음 3개만 보여주고 나머지는 **"+N개 더보기"/"접기"**로 펼칠 수 있습니다. (v2.28.3에서 고친 인증서 카드와 동일한 방식)
- **접근 목록 카드 — 항목 많을 때 스크롤**: 접근 목록을 펼쳤을 때 IP/CIDR 규칙이 많으면 카드가 끝없이 길어지던 것을, **카드 안에서 스크롤**되도록 바꿨습니다.
- **접근 목록 조회 견고성**: 설명(description)이 비어 있는(NULL) 접근 목록·규칙이 있을 때 목록 조회가 실패(500)하던 잠재 문제를 수정했습니다. (정상 경로로는 거의 발생하지 않지만, 수동 편집/구버전 데이터에서 발생 가능)
