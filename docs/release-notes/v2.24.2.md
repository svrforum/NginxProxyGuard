## ⚡ 로그 화면 로딩 속도 대폭 개선 (대용량 환경)

로그가 많이 쌓인 환경에서 **로그 화면 로딩이 매우 느리던 문제**를 개선했습니다.

- **로그 목록 정렬 방식 최적화** — 로그 목록을 불러올 때 정렬 기준을 데이터베이스 파티션 키(`created_at`)에 맞춰, TimescaleDB가 최신 데이터부터 필요한 만큼만 읽고 멈추도록 했습니다. 수천만~수억 건이 쌓인 환경에서 목록 로딩이 **수 초 → 수십 ms 수준**으로 빨라집니다(표시 순서는 동일).
- 페이지 이동(다음 페이지)도 같은 방식으로 처리해 깊은 페이지에서도 빠릅니다.

> 참고: 오래된 버전(예: v2.18.x)에서 올라오시는 경우, 이 버전에 포함된 기본 조회 기간 제한·통계 캐시까지 함께 적용되어 로그/통계 화면 체감 속도가 크게 개선됩니다. 통계(차트) 집계의 추가 최적화는 이후 버전에서 이어집니다.
