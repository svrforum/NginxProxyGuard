## 🐛 로그/시스템 로그 안정성 수정

시스템 로그 화면에 보이던 불필요한 "DB 에러"와 일부 잘못된 통계를 바로잡았습니다.

- **시스템 로그의 가짜 DB 에러 제거** — 로그 필터(로그 유형·심각도·차단 사유·소스·레벨, 프록시 호스트/사용자 ID)에 잘못된 값이 들어오면 데이터베이스까지 전달돼 오류로 기록되던 문제를 수정했습니다. 이제 잘못된 필터 값은 안전하게 무시됩니다.
- **재시작 메시지를 에러로 표시하지 않음** — 컨테이너 재시작 시 데이터베이스가 남기는 정상 종료 메시지(예: "terminating ... due to administrator command")가 치명적 오류처럼 보이던 것을, 정상 동작 로그로 분류하도록 변경했습니다.
- **시스템 로그 소스 필터 정상화** — `audit`·`api_token` 소스로 필터링이 정상 동작하도록 바로잡았습니다.
- **GeoIP 국가 통계 중복 집계 수정** — 같은 국가가 여러 줄로 나뉘어 합계가 부풀려지던 문제를 수정했습니다.

> 참고: 로그 화면 로딩 속도(특히 데이터가 많은 운영 환경)는 별도로 조사 중입니다.
