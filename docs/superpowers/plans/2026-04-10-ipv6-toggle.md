# IPv6 Listen Toggle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a global setting to enable/disable IPv6 listen directives (`listen [::]:`) in all generated nginx configs, so servers with IPv6 disabled can run NPG without manual config editing.

**Architecture:** Add `enable_ipv6` boolean (default `true`) to `global_settings` DB table and model. The nginx config templates conditionally include `listen [::]:` lines based on this setting. When toggled, all configs (proxy hosts, redirects, default server) are regenerated and nginx is reloaded.

**Tech Stack:** Go 1.24 (Echo v4), PostgreSQL 17, React 18 + TypeScript, i18next
